## 2026-04-09 - Fixed N+1 Query via Preload Wrapper in Workspace Member Retrieval
**Learning:** Found an N+1 query issue in the workspace member API (`src/workspace/handler/member.go`) where policies were being queried individually for each member within a loop. The core entity `WorkspaceMember` from `wacraft-core` lacked a `Policies` relationship, preventing direct use of `.Preload("Policies")`. Additionally, temporary `main` package files can break GitHub Actions CI scans (CodeQL).
**Action:** When working with external GORM entities that lack relationships, declare a local wrapper struct that embeds the core entity and adds the missing slice with a `gorm:"foreignKey:..."` tag. This allows using `.Model(&CoreEntity{}).Preload("MissingRelation").Find(&wrapperSlice)`. Ensure temporary code files are removed before submitting.
